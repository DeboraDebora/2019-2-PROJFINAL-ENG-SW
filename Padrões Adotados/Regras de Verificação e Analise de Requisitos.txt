Requisitos de um sistema são as
descrições do que o sistema deve
fazer, os serviços que oferece e as
restrições a seu funcionamento. Eles
refletem as necessidades dos clientes
para um determinado sistema”.

No caso deste sistema que está sendo desenvolvido neste trabalho, devem-se atender a Requisitos Funcionais (RF) e Requisitos Não-Funcionais (RNF). 

Requisitos Funcionais: Definem O QUE O SISTEMA fará. Em outras palavras, o sistema deve receber as credenciais (login e senha) de usuários cadastrados (ou então permitir que eles se cadastrem), para então possibilitar que eles façam login, criem um perfil e o consultem, alterem ou excluam. 
Requisitos Não-Funcionais: são praticamente todas as necessidades que não podem ser atendidas através de funcionalidades. Geralmente mensurável, os requisitos não funcionais definem características e impõe limites do sistema como método de desenvolvimento, tempo, espaço, Sistema Operacional, dentre outros. Neste caso, é necessário que seja um sistema fácil de ser utilizado por usuários com pouca habilidade em informática, além de estar disponível 99% do tempo. 

Além disso, deve atender a estas três regras de requisitos listadas abaixo:

Requisitos de Usuário
Um usuário (prestador ou recebedor de serviços) deve ser capaz de se cadastrar no sistema.
Deve ser capaz de fazer login no sistema.
Deve ser capaz de criar um perfil de Usuario (com um formulário a ser preenchido), consultar seu perfil de usuário, fazer alterações neste perfil, excluir usuário (excluir conta). 

O administrador do Sistema deve ser capaz de consultar usuário, excluir usuário, dar suporte aos usuários.

Requisitos de Sistema 
O sistema deve proporcionar formulário de cadastro e de login, permitir que o usuário crie um perfil com seus dados, alterá-lo, exclui-lo quando desejar. 

Requisitos de Processo 
Linguagens HTML/CSS/PHP e o banco de dados SQL. 

Requisitos de ambiente: possível de acessar em qualquer computador ou celular, inclusive por usuários com deficiência visual e/ou intelectual / neurológica. Portabilidade de sistemas operacionais.
